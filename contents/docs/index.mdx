## What is Fhish?

Fhish is a high-performance privacy layer built for the **Initia** network. By combining the hybrid Cosmos/EVM architecture of Initia with **Fully Homomorphic Encryption (FHE)**, Fhish enables the creation of private rollups where smart contracts can perform computations on encrypted data without ever decrypting it.

At its core, Fhish provides the infrastructure to build **Privacy-First Rollups**.

### The Fhish Stack

1.  **Fhish-CLI** - A native Go orchestration tool for managing your private rollup lifecycle.
2.  **FHE-Gateway** - An independent decryption service that handles secure re-encryption and tallying of FHE results.
3.  **FHE-Relayer** - An automated service that bridges FHE computation requests between the rollup and the gateway.
4.  **fhish-wasm** - A core Rust library (compiled to WASM) that provides the underlying TFHE primitives.

## Why Fhish?

### 1. True Privacy
Unlike Zero-Knowledge (ZK) proofs which prove the validity of a computation, FHE allows the **computation itself** to remain private. This means you can have a private voting contract where the tally is encrypted and only the final result is revealed.

### 2. Seamless DX
Developers can write standard Solidity contracts that interact with FHE precompiles. Our CLI handles the complex orchestration of nodes, keys, and relayers.

### 3. Native Initia Integration
Fhish is built specifically for Initia's **Interwoven Stack**, leveraging MiniEVM for high throughput and IBC compatibility.
